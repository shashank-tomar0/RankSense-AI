# Contributing to TalentScout AI

First off, thank you for considering contributing to TalentScout AI! It's people like you that make the open-source community such a powerful place to build and innovate.

## 🚀 How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please create an Issue on GitHub. Include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- Expected vs. actual behavior.
- System information (OS, Python version, Node version).

### 2. Suggesting Enhancements
We are always looking for ways to improve our 85+ forensic features. If you have an idea:
- Check existing issues to see if it's already been suggested.
- Open a new Issue describing the feature, why it's needed, and how it aligns with our goal of "Forensic Recruitment."

### 3. Submitting Pull Requests
1. **Fork the repo** and create your branch from `main` (`git checkout -b feature/AmazingFeature`).
2. **Install dependencies**: Make sure you have the required Python and Node environments set up as per the `README.md`.
3. **Commit your changes**: Write clear, concise commit messages (`git commit -m 'Add some AmazingFeature'`).
4. **Push to the branch** (`git push origin feature/AmazingFeature`).
5. **Open a Pull Request**: Describe your changes in detail and link to any relevant Issues.

## 🛠️ Development Guidelines
- **Code Style**: We follow standard Python (PEP 8) and standard React/Next.js formatting conventions. Please run your code through standard linters before submitting.
- **Testing**: If you add new AI extraction logic or forensic features, please ensure existing tests pass and consider adding new tests for your specific logic.

We review pull requests on a rolling basis. Thank you for helping us bring absolute truth and transparency back to hiring!
